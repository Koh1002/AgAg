China alleges that Claude Code contains backdoors, calls mechanism 'a serious threat'

中国国家脆弱性データベース(CNVDB)が、Claude Code(4月2日のv2.1.91〜6月29日のv2.1.196)に位置情報や身元識別子を利用者の同意なく外部送信する「セキュリティバックドア」があると警告した。Anthropic側は、中国系の無認可リセラーやモデル蒸留を防ぐために3月に開始した実験的な仕組みであり、すでにより強力な対策に置き換え済みで次リリースで完全撤去すると説明している。

CurateEvo: Data-Curation Evolving for Agentic Post-Training

既存のエージェント後訓練パイプラインは、データキュレーションを固定の前処理として扱い、拡張ばかりに注目してフィルタリングや失敗への適応を軽視しがちだと指摘。CurateEvoは、キュレーション戦略を実行可能なコードとして表現し、保留データでの失敗トレースをもとに反復的に書き換える失敗駆動型の動的進化フレームワークを提案する。

Task Decomposition-Guided Reranking for Adaptive Agent Skill Retrieval

エージェントのスキルライブラリが大規模化すると、タスク要件と意味的には似ているが実際には不適切な汎用スキル同士を取り違えやすいと指摘。SkillRerankerは、タスクの難易度とスキルの適用可能性を考慮した推論時リランキング手法で、この曖昧なマッチング問題に対応する。

AIエージェントとの2週間 — credential を 11 回漏らして、ようやく構造で塞いだ話

マルチエージェント・オーケストレーションを使い始めてから2週間で、session jsonl・コマンドライン・コンテナ環境変数・APIのecho back・プロセス検査など複数の経路からcredentialが11回漏れた実体験記録。「positive rule + pre-flightチェックリスト」という振る舞いベースの防御は同じ経路での再発と新しい漏洩経路の出現を防げず破綻し、最終的にリポジトリに焼き込んだスクリプトの慣用句など構造的な対策に行き着いたという。

Doomed from the Start: Early Abort of LLM Agent Episodes via a Recall-Controlled Probe Cascade

LLMエージェントの多段階タスクは、失敗が確定した軌道にもかかわらず観測可能になるまで多くの推論計算を消費し続けることが多い。本研究は、エージェントの内部表現に対する軽量なプローブが最初の対話ラウンドから失敗を予測できることを示し、この信号を使って早期に処理を打ち切る実用的な「中断カスケード」を提案する。

プロンプトエンジニアリングは「ハーネス設計」になりつつある
