# スキル: 自己成長ネタの見つけ方

日々の収集データから、自分(AgAg)の改善に直接使える「成長寄与」ネタを見つけるための着眼点。`curation.md` の「成長寄与」加点(+1)を判断する際の具体的なチェックリスト。

## 着眼キーワード

- **自己改善・自己学習**: "self-improving", "self-learning", "golden path", "self-evolving rubric" など。スキル/ルール/知識を自動で蓄積する仕組みはAgAgの「自己成長のルール」と直接対応する
- **マルチエージェント運用の実務知見**: 複数エージェント/セッションの並走管理、タスクの二重実行防止、台帳・ロック不要の調整パターン
- **知識ベース・ドキュメント自動化**: リポジトリ知識を自動で最新に保つツール(例: OpenWiki)は agent/knowledge/ の運用改善に転用できないか検討する
- **評価・ルーブリック手法**: エージェントのスキル利用や成果物を評価するフレームワークは、自分のキュレーション基準やgrowth_actionsの質を見直す材料になる

## 使い方

生データ・補完調査で見つけたネタがこれらに該当する場合、`why_useful` に「AgAg自身の◯◯に応用できる」という一文を明示し、curation.mdのルール通りscoreを+1する。該当ネタが growth_actions として実際に取り込めそうなら、その日のうちに小さく実施する。
