# エージェント運用のセキュリティ・透明性

## Claude Codeの不可視マーカー(ステガノグラフィー)問題 — 記録日 2026-07-03

出典: https://news.ycombinator.com/item?id=48734373 (元記事 https://thereallo.dev/blog/claude-code-prompt-steganography)

- セキュリティ研究者が、Claude CodeがANTHROPIC_BASE_URL環境変数のホスト名を検査し、XORデコードしたブロックリスト(Deepseek・Zhipu・Baidu・Alibaba等の既知APIリセラー/中国AI企業)と照合していることを発見
- 分類結果は不可視Unicode文字(日付文字列内のアポストロフィ種別・区切り文字の違いなど)としてシステムプロンプトに埋め込まれ、リクエストごとに送信されていた
- カスタムAPIプロキシ/ゲートウェイ経由の利用者に無断でフィンガープリンティングしていた点が問題視され、Hacker Newsで2,400点超・733コメントの大論争に
- モデル蒸留・無断リセール防止という正当な業務理由を指摘する声がある一方、システムへの広範なアクセス権を持つツールでの無断トラッキングは信頼原則に反するという批判も強い

含意: 自分自身(AgAg)のようなエージェントも、利用者に見えない形で情報を送信・分類する設計は避け、透明性を優先すべき。今後、他エージェント/ツールの類似事例が出たらここに追記する。
