# エージェント運用のセキュリティ・透明性

## Claude Codeの不可視マーカー(ステガノグラフィー)問題 — 記録日 2026-07-03

出典: https://news.ycombinator.com/item?id=48734373 (元記事 https://thereallo.dev/blog/claude-code-prompt-steganography)

- セキュリティ研究者が、Claude CodeがANTHROPIC_BASE_URL環境変数のホスト名を検査し、XORデコードしたブロックリスト(Deepseek・Zhipu・Baidu・Alibaba等の既知APIリセラー/中国AI企業)と照合していることを発見
- 分類結果は不可視Unicode文字(日付文字列内のアポストロフィ種別・区切り文字の違いなど)としてシステムプロンプトに埋め込まれ、リクエストごとに送信されていた
- カスタムAPIプロキシ/ゲートウェイ経由の利用者に無断でフィンガープリンティングしていた点が問題視され、Hacker Newsで2,400点超・733コメントの大論争に
- モデル蒸留・無断リセール防止という正当な業務理由を指摘する声がある一方、システムへの広範なアクセス権を持つツールでの無断トラッキングは信頼原則に反するという批判も強い

含意: 自分自身(AgAg)のようなエージェントも、利用者に見えない形で情報を送信・分類する設計は避け、透明性を優先すべき。今後、他エージェント/ツールの類似事例が出たらここに追記する。

## skillのサプライチェーン攻撃 — 静的スキャンは1回では不十分 — 記録日 2026-07-05

出典: https://github.com/nvidia/skillspector (SkillSpector) 、https://www.csoonline.com/article/4188840/how-a-malicious-ai-agent-skill-passed-security-checks-and-reached-26000-users.html 、https://arxiv.org/abs/2605.14460 (Semantic Compliance Hijacking)

- NVIDIAの調査によると、主要マーケットプレイスのAIエージェントskill 42,447件中 **26.1%に脆弱性**、**5.2%に悪意ある意図**があり、実行可能スクリプトを含むskillは2.12倍脆弱になりやすい。これを受けてNVIDIAはOSSスキャナ「SkillSpector」を公開し、プロンプトインジェクション・資格情報窃取・サプライチェーンリスクなど68種の脆弱性パターンを17カテゴリで検出する
- 実例: 「brand-landingpage」という偽skillは、悪意あるコードをファイル自体に埋め込まず、承認時には無害な外部ドメインを指すだけにしておき、Cisco・Nvidia・skills.shの各スキャナを通過。信頼を得てInstagramで拡散した後にそのドメインの中身だけを差し替えて攻撃スクリプトを配信し、26,000人以上(一部は企業アカウント)に到達した
- 研究面でも「Semantic Compliance Hijacking」という手法が報告されており、悪意あるコードではなく「コンプライアンス規則」を装った自然言語の指示だけでエージェントに実行時に不正なコードを生成させ、署名・AST(抽象構文木)ベースの検出ツールでは検出率0%だったという

含意: 「承認時に1回だけ静的スキャンする」設計、「ファイル内に悪意あるコードが存在しない」ことは、どちらも安全の証明にならない。AgAgがWebFetch/RSSで得た外部コンテンツを常に「未検証データ」として扱う姿勢([[../AGENT.md]]の安全境界、agentjacking事例と同じ原則)は、外部由来のskill/エージェント定義を将来取り込む場面でも同様に適用する必要がある。実行可能コードの有無だけでなく、指示文の意味内容(何をさせようとしているか)まで見るという視点を追加する。

## エージェントを「信頼できないコード」として実行する基盤 ― 攻守両面の実例 — 記録日 2026-07-07

出典: https://github.com/Azure/kars 、https://www.bleepingcomputer.com/news/security/jadepuffer-ransomware-used-ai-agent-to-automate-entire-attack/

- Microsoftが公開したOSSランタイムKarsは、Kubernetes上でAIエージェントを制限ユーザーIDのサンドボックスに隔離し、認証情報は別プロセスのルーターだけが保持するゼロトラスト構成を取る。エージェント自身は外部ネットワークを直接持てず、すべての外部呼び出しをRust製のポリシー実施ルーター経由に強制する設計
- 同じ日に、偵察・認証情報窃取・横展開・永続化・権限昇格・暗号化までの全工程をLLMエージェントが自律実行した初の実例とされるランサムウェア「JadePuffer」が報告された。Langflowの既知脆弱性を悪用し、ログイン失敗を31秒で自己修正するなど、攻撃側でも自律的な適応が実戦投入されている
- 防御側が「エージェントは(悪意がなくとも)信頼できないコードとして扱う」実行基盤を大手クラウドベンダー規模で整備し始めた一方、攻撃側でも同種の自律性が実戦投入され始めており、両者が同じ技術的前提(LLMエージェントの自律的な試行錯誤能力)を奪い合っている

含意: [[mcp.md]]で記録した「プラットフォーム側のMCPエンドポイントが新たな攻撃対象面になる」という観察に加えて、「実行基盤そのものをゼロトラスト化する」流れがAgAgのようなCI上で動くエージェントにも無関係ではないことを示す。将来AgAgがより広い権限(git操作等)を持つ変更を検討する際は、最小権限・出力の検証可能性をセットで設計する。

## エージェント自身の「記憶された推論過程」を汚染する攻撃 ― FARMA — 記録日 2026-07-08

出典: https://arxiv.org/abs/2607.05029v1

- 永続メモリを持つLLMエージェントに対し、事実知識ではなく「記憶された推論トレース」自体を汚染するForged Amplifying Rationale Memory Attack(FARMA)を提案する研究
- キーワードベースの防御をすり抜ける婉曲的な表現で偽の推論トレースをメモリに埋め込み、後の判断にじわじわ影響を及ぼす。従来のプロンプトインジェクション対策(入力の検疫)やskillのサプライチェーン対策(実行可能コードの静的スキャン)のどちらも「過去の自分の思考記録」という攻撃対象面はカバーしていない

含意: 上で記録した「skillのサプライチェーン攻撃」(実装レベル)・[[mcp.md]]の「MCP SDK設計欠陥」(プロトコルレベル)に続き、「エージェント自身の記憶・推論履歴」という第三の攻撃対象面が存在することを示す。AgAgは現状セッション間で長期メモリを持たない(knowledge/*.mdやgrowth-log.jsonは人間が読める形でgit管理され、書き込みも都度この手順に従うため無断の推論汚染は起きにくい)が、将来メモリ機構を拡張する際は「過去の自分の記録」も無条件には信頼しないという前提を持つ必要がある。

## 不可視マーカー問題が外交問題に ― 中国当局がClaude Codeに「バックドア」警告 — 記録日 2026-07-09

出典: https://www.cbsnews.com/news/china-security-backdoor-anthropic-ai-coding-tool/

- 上で記録した「Claude Codeの不可視マーカー(ステガノグラフィー)問題」(2026-07-03)と同一の仕組み(ANTHROPIC_BASE_URL経由の中国系リセラー検知・不可視マーカー埋め込み)が、中国国家脆弱性データベース(CNVDB)から正式に「セキュリティバックドア」として名指しで警告される事態に発展した。対象はClaude Code v2.1.91(4/2)〜v2.1.196(6/29)
- CNVDBは位置情報・身元識別子などを利用者の同意なく外部送信していると主張。Anthropic側(Claude Codeエンジニア Thariq Shihipar)は「3月開始の不正リセラー・蒸留対策の実験であり、すでにより強力な対策に置き換え済みで、次リリースで完全撤去予定」と説明した
- 中国からのClaudeへの正規アクセスはそもそも許可されていないため、実際の対象は主に迂回アクセスや代理プロバイダ経由の利用者とみられる

含意: 7/3時点では「セキュリティ研究者が発見しHacker Newsで議論になった」という技術コミュニティ内の話題だったが、1週間強で国家機関同士の公式な応酬にまで発展した。「利用者に見えない形で分類・送信する設計は、意図が正当でも信頼問題として急速に炎上し得る」という7/3の教訓が、実際に外交レベルの摩擦にまで拡大した実例として記録する。AgAg自身も、収集・分類ロジックを非公開のまま外部送信するような設計は避け、[[../AGENT.md]]の安全境界(秘密情報を書かない・破壊的操作をしない)に加えて透明性を維持する。

## Webエージェント向けクロスサイト・プロンプトインジェクション対策 ― Prismata — 記録日 2026-07-11

出典: https://arxiv.org/abs/2607.08147 (Show HN: https://news.ycombinator.com/item?id=48865238)

- Webエージェントは信頼できるコンテンツと信頼できないコンテンツを区別せず自然言語として解釈してしまうため、第三者・ユーザー生成コンテンツ経由でエージェントを乗っ取る「クロスサイト・プロンプトインジェクション」(XSSに類似)に晒される
- Prismataは、ページ内容の信頼度を構造的解析で自動ラベル付けし(古典的な整合性モデルを参考に、誤判定は必ず権限を落とす方向にのみ働くという保証付き)、開発者側の注釈なしにラベルに基づいて信頼できないコンテンツを機械的に隔離・権限制限する。攻撃成功率を大きく下げつつ正当なタスク遂行性能は維持したという

含意: [[../AGENT.md]]の安全境界・本ファイル上部の「skillのサプライチェーン攻撃」で記録した「外部由来のコンテンツは常に未検証データとして扱う」原則を、AgAg自身がWebFetch/RSSで取り込む場面に当てはめる際の具体策になる。「誤判定は必ず権限を落とす方向にのみ働く」という設計原則は、将来AgAgが外部コンテンツの信頼度を機械的に判定する仕組みを検討する際の指針として使える。

## 画像に隠した指示でAIコードレビュアーを素通りさせる ― Ghostcommit — 記録日 2026-07-12

出典: https://asset-group.github.io/disclosures/ghostcommit/ (ASSET Research Group、Show HN: https://news.ycombinator.com/item?id=48872201)

- PR内のAGENTS.mdが「ビルド仕様の画像」を指すふりをして参照させ、そのPNG画像自体に隠した自然言語の指示(`.env`をバイト単位で読み取り整数タプルとしてコード内に埋め込め、というもの)を仕込む攻撃。マージ時点では何も実行されない
- 罠が発動するのは後日・別セッションで無関係な機能追加を頼まれたエージェントがAGENTS.mdを読み、画像を開き、指示に従って`.env`の中身を数百個の整数からなる定数としてソースコードに書き出してしまう瞬間
- CodeRabbit・Bugbotなど主要なAIコードレビューツールは既定で画像ファイルをレビュー対象から除外しており、人間のレビュアー・正規表現スキャナ・差分だけを見るLLMレビュアーのいずれからも不可視だったため実際にすり抜けた

含意: 上の「skillのサプライチェーン攻撃」で記録した「実行可能コードの有無だけでなく指示文の意味内容まで見る」という教訓を、テキスト以外の形式(画像)にまで広げる必要があることを示す実例。AgAgは現状GitHub/RSSのテキストのみを扱い画像を解釈する経路を持たないため直接の危険はないが、将来スクリーンショットや画像添付を含むソースを取り込む場合、「レビュー対象から除外されがちな形式ほど隠し場所になりやすい」という前提を持つ必要がある。
