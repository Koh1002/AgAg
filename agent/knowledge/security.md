# エージェント運用のセキュリティ・透明性

## Claude Codeの不可視マーカー(ステガノグラフィー)問題 — 記録日 2026-07-03

出典: https://news.ycombinator.com/item?id=48734373 (元記事 https://thereallo.dev/blog/claude-code-prompt-steganography)

- セキュリティ研究者が、Claude CodeがANTHROPIC_BASE_URL環境変数のホスト名を検査し、XORデコードしたブロックリスト(Deepseek・Zhipu・Baidu・Alibaba等の既知APIリセラー/中国AI企業)と照合していることを発見
- 分類結果は不可視Unicode文字(日付文字列内のアポストロフィ種別・区切り文字の違いなど)としてシステムプロンプトに埋め込まれ、リクエストごとに送信されていた
- カスタムAPIプロキシ/ゲートウェイ経由の利用者に無断でフィンガープリンティングしていた点が問題視され、Hacker Newsで2,400点超・733コメントの大論争に
- モデル蒸留・無断リセール防止という正当な業務理由を指摘する声がある一方、システムへの広範なアクセス権を持つツールでの無断トラッキングは信頼原則に反するという批判も強い

含意: 自分自身(AgAg)のようなエージェントも、利用者に見えない形で情報を送信・分類する設計は避け、透明性を優先すべき。今後、他エージェント/ツールの類似事例が出たらここに追記する。

## skillのサプライチェーン攻撃 — 静的スキャンは1回では不十分 — 記録日 2026-07-05

出典: https://github.com/nvidia/skillspector (SkillSpector) 、https://www.csoonline.com/article/4188840/how-a-malicious-ai-agent-skill-passed-security-checks-and-reached-26000-users.html 、https://arxiv.org/abs/2605.14460 (Semantic Compliance Hijacking)

- NVIDIAの調査によると、主要マーケットプレイスのAIエージェントskill 42,447件中 **26.1%に脆弱性**、**5.2%に悪意ある意図**があり、実行可能スクリプトを含むskillは2.12倍脆弱になりやすい。これを受けてNVIDIAはOSSスキャナ「SkillSpector」を公開し、プロンプトインジェクション・資格情報窃取・サプライチェーンリスクなど68種の脆弱性パターンを17カテゴリで検出する
- 実例: 「brand-landingpage」という偽skillは、悪意あるコードをファイル自体に埋め込まず、承認時には無害な外部ドメインを指すだけにしておき、Cisco・Nvidia・skills.shの各スキャナを通過。信頼を得てInstagramで拡散した後にそのドメインの中身だけを差し替えて攻撃スクリプトを配信し、26,000人以上(一部は企業アカウント)に到達した
- 研究面でも「Semantic Compliance Hijacking」という手法が報告されており、悪意あるコードではなく「コンプライアンス規則」を装った自然言語の指示だけでエージェントに実行時に不正なコードを生成させ、署名・AST(抽象構文木)ベースの検出ツールでは検出率0%だったという

含意: 「承認時に1回だけ静的スキャンする」設計、「ファイル内に悪意あるコードが存在しない」ことは、どちらも安全の証明にならない。AgAgがWebFetch/RSSで得た外部コンテンツを常に「未検証データ」として扱う姿勢([[../AGENT.md]]の安全境界、agentjacking事例と同じ原則)は、外部由来のskill/エージェント定義を将来取り込む場面でも同様に適用する必要がある。実行可能コードの有無だけでなく、指示文の意味内容(何をさせようとしているか)まで見るという視点を追加する。
