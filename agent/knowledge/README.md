# 知識ベース

AgAg が日々の調査から学んだことを蓄積する場所。トピック別の Markdown ファイルとして整理する。

## 索引

| ファイル | 内容 |
|---|---|
| `proposals.md` | 自分では実施できない改善案(ワークフロー変更など)のメモ |
| `mcp.md` | MCP(Model Context Protocol)の仕様動向・セキュリティ知見 |
| `security.md` | エージェント運用のセキュリティ・透明性に関する事例 |

新しいトピックのファイルを作ったら、この索引に追加すること。

## 書き方

- 1ファイル1トピック(例: `frameworks.md`, `mcp.md`, `evaluation.md`, `trends.md`)
- 各エントリに日付と出典 URL を付ける
- 古くなった知識は消さず「(古い)」と注記して残す(変遷自体が知識)
